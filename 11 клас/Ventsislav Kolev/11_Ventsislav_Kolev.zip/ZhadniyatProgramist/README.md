# 🍺 Кръчма „Жадният Програмист“ — тефтерите на бай Пешо

Desktop приложение (WinForms, .NET 8, SQLite), което води тефтера на кръчмата:
кой програмист какво е поръчал, колко дължи и кога най-после си плаща.

## Стартиране

1. Отвори `ZhadniyatProgramist.sln` във Visual Studio 2022 (или по-нов).
2. Натисни **F5**.

Това е всичко:
- NuGet пакетът `Microsoft.Data.Sqlite` се възстановява автоматично;
- при първо стартиране се създава `krachma.db` до `.exe` файла,
  заедно с таблиците и примерните данни;
- не е нужен SQL Server, инсталации или настройки.

## Функционалност (непроменена бизнес логика)

- **JOIN таблица** с всички поръчки (имена вместо ID-та) + търсене на живо;
- **SQL SUM** на дълга на избрания програмист;
- **Плащане** — параметризиран `DELETE` на всички поръчки на човека;
- **Добавяне** на поръчки и програмисти с пълна валидация;
- **Статистика**: брой програмисти, активни сметки, общ дълг, най-затънал;
- Всички заявки са **параметризирани** — защита срещу SQL Injection.

## Звуци (незадължителни)

**За да има реални звуци, постави `.wav` файлове в `Assets/Sounds/`
с точните имена:**

`order.wav`, `payment.wav`, `error.wav`, `big_debt.wav`, `select.wav`,
`hover.wav`, `click.wav`, `ambience.wav` (loop, кръчмарска атмосфера).

- Липсва файл → **тишина** (без системните звуци на Windows и без краш).
  Единствено при грешка без `error.wav` има кратък тих beep,
  изпълнен на background нишка.
- `SoundService.Enabled` (checkbox-ът „Звук: Вкл/Изкл“ в статус бара)
  реално спира **всички** звуци, включително атмосферата.
- Файловете се копират автоматично до `.exe` при build (`.csproj`).

## Структура

```
ZhadniyatProgramist.sln
ZhadniyatProgramist.csproj
Program.cs                 — вход: база -> loading -> главен прозорец
DatabaseHelper.cs          — целият достъп до SQLite (НЕ е променян)
MainForm.cs / .Designer.cs — главният екран
LoadingForm.cs / .Designer.cs
Models/                    — Programmer, TabItem
Controls/                  — всички custom контроли (виж таблицата горе)
Services/                  — SoundService, AnimationService
SQL/create_database.sql    — документация на схемата
Assets/Pesho, Assets/Sounds — незадължителни ресурси
```

## Производителност

- Всички анимирани контроли са с `OptimizedDoubleBuffer`;
- фонът се прерисува само в празнините (децата го изрязват);
- всеки ефект е на лек `Timer` (40–100 ms) и `Invalidate` се вика
  само когато има какво да се промени;
- при прозорец под вътрешния минимум се появяват скролери —
  никога изрязани текстове или застъпени контроли.

## Как се тества бързо

1. **F5** → loading екран с shimmer → плавен fade-in на главния прозорец.
2. Избери *Георги Stackoverflow* → дългът „тича“ нагоре, Пешо се ядосва;
   ако е над 100 € — панелът се разклаща, рамката пулсира червено.
3. Добави поръчка на човек с дълг над 50 € → „тук-тук“ +
   „Първо плащаш, после поръчваш!“.
4. Натисни **ПЛАТИ СМЕТКАТА** → монети + „ПЛАТЕНО“, Пешо намига,
   вдига палец и подскача, статистиките се броят плавно надолу.
5. Смени часовника на компютъра след 18:00 → „🌙 Нощ“, по-тъмен фон,
   неонът и табелата „ОТВОРЕНО“ светят по-силно.
6. Смали прозореца → появяват се скролери, нищо не се чупи.
