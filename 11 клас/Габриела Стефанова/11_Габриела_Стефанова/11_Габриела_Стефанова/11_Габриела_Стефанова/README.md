\# Golf IV — Interstellar Charter



Хумористичен сайт за "частен междузвезден чартър", който всъщност показва

астрономическата снимка на деня от НАСА за избрана от потребителя дата в

миналото, преведена на български.



\## Структура



```

index.html  

style.css    

script.js    

```



\## JS логика (script.js)



\### 1. Инициализация на полето за дата



```js

dateInput.max = todayStr;        // не позволява бъдещи дати

dateInput.min = '1995-06-16';    // NASA APOD архивът започва оттук

dateInput.value = '2008-03-01';  // дестинация по подразбиране

```



Валидацията на диапазона се прави \*\*преди\*\* заявката, за да не хабим

извиквания към NASA API за очевидно невалидни дати.



\### 2. Заявка към NASA APOD API



```js

const res = await fetch(

&#x20; `https://api.nasa.gov/planetary/apod?api\_key=${NASA\_API\_KEY}\&date=${chosenDate}`

);

data = await res.json();

```



Особености, с които се съобразява кодът:



\- \*\*NASA не хвърля винаги HTTP грешка\*\* — при достигнат rate limit връща

&#x20; `200 OK`, но тялото съдържа `{ error: { code, message } }`. Затова

&#x20; проверката е:

&#x20; ```js

&#x20; if (data.error || data.code || !data.url) { handleNasaFailure(); return; }

&#x20; ```

&#x20; а не просто `res.ok`.

\- Мрежов/fetch срив (offline, невалиден домейн и т.н.) се хваща отделно

&#x20; с `try/catch` около самия `fetch`, преди да се стигне до JSON парсване.

\- `media\_type` може да е `"image"` или `"video"` (APOD понякога публикува

&#x20; видео) — при видео се ползва `thumbnail\_url` вместо `url`.



\### 3. Fallback / кеш стратегия



При провал (rate limit, мрежа) сайтът не чупи UI-а, а:



1\. Ако вече има заредена и преведена дестинация по подразбиране

&#x20;  (`01.03.2008`, кеширана в `cachedDefaultView`) — връща потребителя към

&#x20;  нея и показва обяснителен `errorBox`.

2\. Ако дори първоначалното зареждане е паднало — показва честна съобщение

&#x20;  за грешка без ticket карта.



Кешът се пълни на два етапа: снимката/датата веднага след успешния NASA

отговор, а `title`/`desc` — след като преводът приключи, за да не се кешира

непреведен текст.



\### 4. Превод чрез MyMemory API



NASA връща `title` и `explanation` на английски. Те се превеждат асинхронно,

\*\*успоредно\*\*, докато потребителят вече вижда снимката с placeholder текст

`"Превежда се..."`:



```js

const \[titleBg, descBg] = await Promise.all(\[

&#x20; translateToBulgarian(data.title),

&#x20; translateToBulgarian(data.explanation)

]);

```



MyMemory ограничава заявките до \~500 символа, затова по-дългият

`explanation` се реже на изречения (`splitIntoChunks`) и се превежда на

части, които после се сглобяват.



MyMemory също не връща HTTP грешка при изчерпан дневен лимит — вместо превод

праща текст, започващ с `"MYMEMORY WARNING..."`. Това се разпознава ръчно

(`isQuotaExceeded`) и се третира като изключение (`MYMEMORY\_QUOTA\_EXCEEDED`),

за да не се покаже боклук вместо превод — вместо това се пада обратно към

кешираната дестинация от 01.03.2008, а ако темата е точно тя — остава

оригиналният английски текст.



\### 5. Форма и flow



```js

form.addEventListener('submit', function (e) {

&#x20; e.preventDefault();

&#x20; loadDestination(dateInput.value);

});



loadDestination(defaultDate); // автоматично зареждане при отваряне

```



Всяко извикване на `loadDestination`:

1\. Валидира датата.

2\. Вика NASA API.

3\. Показва снимка + оригинален текст мигновено, докато превода тече.

4\. Замества текста с превод (или пада обратно при проблем).
